# GitNexus 内部使用指南

欢迎使用团队定制版的 GitNexus 知识图谱引擎！

通过 GitNexus，你可以让大模型（如 Claude Code）全面掌握我们庞大且复杂的内部代码仓库的结构和调用链，而不会因为超出上下文而报错。为降低你电脑的算力负担并保证分析一致性，我们推荐直接接入**内部公共的远端解析服务器**。如有特殊本地化需求，也可以按需执行下文的纯本地离线安装方案也一样好用。

---

## ⚡ 推荐方案：零安装直连公共测试服务器（免算力开销）

我们已经在远端的公用服务器（内网地址根据分配为准，以下假设为 `192.168.1.100`）运行了全量仓库的知识索引。你**无需在本地安装任何额外的 GitNexus 环境**，甚至不必去扫描代码，所有分析和查询压力均转移在服务端集中处理！

### 前置要求
- 你的电脑能够通过 SSH 登陆研发内网服务器 `192.168.1.100`。

### 挂载操作 (一键完成)
在你本地正工作的任何开发根目录下，呼出终端并使用您的 SSH 用户名配置 Claude Code 的 MCP 挂载点：

```bash
claude mcp add gitnexus -- ssh <你的用户名>@192.168.1.100 "cd /远程代码主仓库绝对路径 && npx gitnexus mcp"
```

配置成功后正常启动 `claude`，你的 AI 助手便拥有了超级权限。直接吩咐它 *“帮我在整个群组服务中分析这段接口如果修改，它上下游的爆炸半径是多少”* 即可体验！

---

## 💾 备选方案：离线包本地纯端安装 (适合出差断网或隔离调试)

如果你被隔离在特殊保密网段无法连接服务器，或者你想在自己私有电脑的分支上做独立的深度解析测试。你可以向项目维护负责人索要一份**内置了改动代码的内网离线安装包**（形如 `gitnexus-1.6.2.tgz` 格式的文件）。

### 1. 全局解压与安装
拿到离线 `.tgz` 压缩包后，放置在你的任意目录下，并在你的终端立刻进行全局应用安装：

```bash
# 修改下方路径为你存放 tgz 安装包的实际绝对路径
npm install -g /你的路径/gitnexus-1.6.2.tgz
```
安装结束后，你的开发机环境中即注册了专属于团队版本的 `gitnexus` 组件。

### 2. 扫瞄你的本地目录
进入你需要调理的大型项目根路径，呼唤其建立本地的图谱索引底座（只需要跑一次）：
```bash
gitnexus analyze
```

### 3. 配置智能体绑定
将本地环境中的知识计算库供给给大模型：
```bash
claude mcp add gitnexus -- gitnexus mcp
```
一旦配对成功，你的后续代码询问都会有精确的底层知识图网络结构做响应背书。

---
💡 **关于升级**: 团队如果发布了更高版本的 `.tgz` 离线依赖包，你只需要重新跑一遍上述的离线 `npm install -g` 步骤即刻享受最新升级的代码逻辑。
